<objective>
Build the core trial stack: the app and frontend Docker images, the nginx reverse-proxy
config, and `docker-compose.trial.yml` wiring all seven services together.

This plan resolves the jOOQ codegen blocker (RESEARCH Open Question 1 / Pitfall 1): jOOQ
generated sources live under gitignored `build/` and require a live PostgreSQL to regenerate.
A plain `docker build` has no Docker socket, so codegen cannot run inside the image. The fix
is to commit the generated sources to `app/src/generated/jooq/` and build with `-x
generateJooq`.

Purpose: Delivers SC-1 (`docker compose up` brings up all services) and the wiring that
makes SC-4 (setup wizard accessible) reachable via `http://localhost`.
Output: jOOQ sources committed, app + frontend Dockerfiles, nginx.conf, docker-compose.trial.yml.

DEPENDENCY NOTE: This plan is Wave 2 — it depends on Plan 01 (forwarder + fake-decoder
Dockerfiles must exist for the compose file to reference them) and Plan 02 (seed Dockerfile
must exist for the demo-seed service).
</objective>

    Resolve by generating the jOOQ sources once on the host and committing them to a
    non-gitignored location, then building with `-x generateJooq`.

    STEP 1 — Generate the sources on the host (uses the existing Gradle pipeline that spins
    up its own postgres container):
    ```
    ./gradlew :app:generateJooq
    ```
    This produces `app/build/generated-sources/jooq/` containing the package
    `dev.monkeypatch.rctiming.jooq.generated`.

    STEP 2 — Copy the generated tree to a committed location:
    ```
    mkdir -p app/src/generated/jooq
    cp -r app/build/generated-sources/jooq/. app/src/generated/jooq/
    ```

    STEP 3 — In `app/build.gradle.kts`, point the jOOQ generation `directory` at the new
    committed location AND add it as a source set so the compiler picks it up. Change the
    jOOQ generate target line:
    ```kotlin
    // was:  directory = "build/generated-sources/jooq"
    directory = "src/generated/jooq"
    ```
    Add the directory to the main source set (place near the top-level `java { }` or after
    the `jooq { }` block):
    ```kotlin
    sourceSets["main"].java.srcDir("src/generated/jooq")
    ```
    This guarantees `:app:compileJava` and `:app:bootJar` find the generated classes even
    when `-x generateJooq` skips regeneration.

    STEP 4 — In `.gitignore`, the `build/` and `**/build/` rules already exclude the old
    location. The new path `app/src/generated/jooq/` is NOT under `build/`, so it is tracked
    by default. To make the intent explicit, add a comment line to `.gitignore`:
    ```
    # jOOQ generated sources are committed at app/src/generated/jooq (Docker builds skip codegen)
    ```
    Do NOT add an ignore rule for `app/src/generated/` — it must stay tracked.

    STEP 5 — Verify the generated sources are seen by git:
    `git add app/src/generated/jooq && git status --short app/src/generated/jooq` shows files.

    STEP 6 — Confirm the build still compiles with codegen skipped:
    `./gradlew :app:compileJava -x generateJooq --no-daemon`
  </action>

    ```dockerfile
    # syntax=docker/dockerfile:1
    FROM eclipse-temurin:21-jdk-alpine AS builder
    WORKDIR /build
    COPY gradlew settings.gradle.kts build.gradle.kts ./
    COPY buildSrc/ buildSrc/
    COPY gradle/ gradle/
    COPY app/ app/
    COPY forwarder/ forwarder/
    RUN --mount=type=cache,target=/root/.gradle \
        chmod +x gradlew && \
        ./gradlew :app:bootJar -x test -x generateJooq --no-daemon

    FROM eclipse-temurin:21-jre-alpine AS runtime
    RUN apk add --no-cache curl
    WORKDIR /app
    COPY --from=builder /build/app/build/libs/app-*.jar app.jar
    EXPOSE 8080 9090
    ENTRYPOINT ["java", "-jar", "app.jar"]
    ```

    Both `app/` and `forwarder/` are copied because the build is multi-project. `buildSrc/`
    holds precompiled script plugins — required. `curl` in the runtime stage is required by
    the compose healthcheck (`curl -sf http://localhost:8080/actuator/health`).

    Create `docker/nginx/nginx.conf` — a full nginx config (replaces the default). Use the
    RESEARCH.md Pattern 2 example. Non-negotiable elements:
    - top-level `worker_processes auto;` and `events { worker_connections 1024; }`
    - inside `http { }`: `include mime.types;`, `default_type application/octet-stream;`,
      `sendfile on;`, `gzip on;` with `gzip_types text/plain text/css application/json
      application/javascript;`
    - the WebSocket upgrade map:
      ```nginx
      map $http_upgrade $connection_upgrade {
          default upgrade;
          ''      close;
      }
      ```
    - one `server { listen 80; server_name _; }` block containing:
      - `root /usr/share/nginx/html; index index.html;`
      - `location / { try_files $uri $uri/ /index.html; }`  (SPA fallback for React Router)
      - `location /assets/ { expires 1y; add_header Cache-Control "public, immutable"; }`
      - `location /api/ { proxy_pass http://app:8080; proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; }`
      - `location /ws { proxy_pass http://app:8080; proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_set_header Host $host; proxy_read_timeout 3600s; proxy_send_timeout 3600s; }`

    The `proxy_read_timeout 3600s` on `/ws` is required — the nginx default 60s drops
    long-lived STOMP connections mid-race. `try_files ... /index.html` is required or
    deep-linked React Router URLs (e.g. `/admin/events/123`) 404.

    NOTE on MinIO: Plan 04 documents `STORAGE_PUBLIC_BASE_URL`. If logo images are served
    through nginx, add a `location /storage/ { proxy_pass http://minio:9000/; }` block too.
    The trial sets `STORAGE_PUBLIC_BASE_URL=http://localhost/storage/rctiming` so include
    this `/storage/` proxy location (trailing slash on proxy_pass strips the `/storage`
    prefix).

|----------|-------------|
| host -> nginx (:80) | Only externally exposed surface; all other services internal-bridge only |
| nginx -> app (:8080, :9090) | Reverse-proxied API + WebSocket; internal bridge |
| browser WebSocket -> nginx -> app | Long-lived STOMP connections |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-10-01 | Elevation of Privilege | postgres / gRPC / piper / minio host exposure | mitigate | No host port mappings for postgres, app, piper, minio, fake-decoder — only `frontend` (nginx:80) is exposed (D-08); enforced by acceptance criteria |
| T-10-02 | Information Disclosure | default secrets in compose env | mitigate | All secrets use `${VAR:-default}` so .env overrides them; Plan 04 .env.example documents "change before production". JWT_SECRET default is the existing dev value |
| T-10-09 | Denial of Service | nginx dropping STOMP mid-race | mitigate | `/ws` location uses `proxy_read_timeout 3600s` — long-lived WebSocket connections survive idle periods |
</threat_model>
