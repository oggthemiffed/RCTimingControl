  <action>
    Create `.github/workflows/publish-trial-images.yml`. This is the repo's first GitHub
    Actions workflow — `.github/workflows/` may not exist yet; create the directory.

    Trigger ONLY on `v*` version tags (D-09):
    ```yaml
    name: Publish Trial Images
    on:
      push:
        tags: ['v*']
    ```

    One job, matrix over the four images. The job needs `packages: write` to push to GHCR:
    ```yaml
    jobs:
      build-and-push:
        runs-on: ubuntu-latest
        permissions:
          contents: read
          packages: write
        strategy:
          matrix:
            include:
              - image: app
                dockerfile: docker/app/Dockerfile
              - image: frontend
                dockerfile: docker/frontend/Dockerfile
              - image: forwarder
                dockerfile: docker/forwarder/Dockerfile
              - image: fake-decoder
                dockerfile: docker/fake-decoder/Dockerfile
        steps:
          - uses: actions/checkout@v4

          - uses: docker/login-action@v3
            with:
              registry: ghcr.io
              username: ${{ github.actor }}
              password: ${{ secrets.GITHUB_TOKEN }}

          - uses: docker/metadata-action@v5
            id: meta
            with:
              images: ghcr.io/oggthemiffed/rctimingcontrol/${{ matrix.image }}
              tags: |
                type=semver,pattern={{version}}

          - uses: docker/build-push-action@v6
            with:
              context: .
              file: ${{ matrix.dockerfile }}
              push: true
              tags: ${{ steps.meta.outputs.tags }}
              labels: ${{ steps.meta.outputs.labels }}
              cache-from: type=gha
              cache-to: type=gha,mode=max
    ```

    Notes:
    - `context: .` (repo root) for ALL four images — every Dockerfile expects the multi-
      project repo root as build context (the frontend Dockerfile copies `frontend/`, the
      Java ones copy `app/`+`forwarder/`).
    - `docker/metadata-action` with `type=semver,pattern={{version}}` turns a `v1.2.3` tag
      into the image tag `1.2.3` — a pinned semantic version, NO `latest` (D-09).
    - `cache-from/cache-to: type=gha` uses the GitHub Actions build cache.
    - the seed image is intentionally NOT in the matrix — it is built locally by both
      compose files.
    - Use the action versions above (checkout@v4, login-action@v3, metadata-action@v5,
      build-push-action@v6). If a newer major version is the current marketplace stable at
      execution time, prefer it — but these are acceptable.
  </action>
