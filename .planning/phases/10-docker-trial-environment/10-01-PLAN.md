<objective>
Make the forwarder process Docker-deployable and package the fake decoder as a container.

The forwarder reads `forwarder.properties` from the classpath with `localhost` defaults — wrong for Docker bridge networking. This plan adds a one-line `--config-file` argument to `ForwarderApplication.main()` so an entrypoint script can write a correct properties file from env vars before launch. It also packages `SimulatorMain` (generative mode) as the `fake-decoder` container.

Purpose: Wave 1 foundation — the trial compose stack (Plan 03) cannot wire the forwarder or fake-decoder services until these images exist.
Output: One Java change + test, plus `docker/forwarder/` and `docker/fake-decoder/` Dockerfiles and shell scripts.
</objective>

    static ForwarderConfig resolveConfig(String[] args) throws java.io.IOException {

  <action>
    Create `docker/forwarder/Dockerfile` — a multi-stage build with repo root as build context:

    ```dockerfile
    # syntax=docker/dockerfile:1
    FROM eclipse-temurin:21-jdk-alpine AS builder
    WORKDIR /build
    COPY gradlew settings.gradle.kts build.gradle.kts ./
    COPY buildSrc/ buildSrc/
    COPY gradle/ gradle/
    COPY app/ app/
    COPY forwarder/ forwarder/
    RUN --mount=type=cache,target=/root/.gradle \
        chmod +x gradlew && \
        ./gradlew :forwarder:installDist -x test --no-daemon

    FROM eclipse-temurin:21-jre-alpine AS runtime
    WORKDIR /app
    COPY --from=builder /build/forwarder/build/install/forwarder/ .
    COPY docker/forwarder/entrypoint.sh /entrypoint.sh
    RUN chmod +x /entrypoint.sh
    ENTRYPOINT ["/entrypoint.sh"]
    ```

    Both `app/` and `forwarder/` must be copied because `settings.gradle.kts` declares a
    multi-project build (`include(":app", ":forwarder")`); Gradle resolves the whole build
    from root. `buildSrc/` contains precompiled script plugins — omitting it breaks the build.

    Create `docker/forwarder/entrypoint.sh` (POSIX sh — image base is Alpine):

    ```sh
    #!/bin/sh
    # Writes forwarder.properties from env vars, then launches the forwarder.
    set -e
    cat > /tmp/forwarder.properties <<PROPS
    forwarder.api-token=${FORWARDER_API_TOKEN}
    forwarder.decoder.host=${FORWARDER_DECODER_HOST:-fake-decoder}
    forwarder.decoder.port=${FORWARDER_DECODER_PORT:-5100}
    forwarder.grpc.host=${FORWARDER_GRPC_HOST:-app}
    forwarder.grpc.port=${FORWARDER_GRPC_PORT:-9090}
    forwarder.grpc.plaintext=${FORWARDER_GRPC_PLAINTEXT:-true}
    PROPS
    exec /app/bin/forwarder --config-file=/tmp/forwarder.properties
    ```

    `exec` is required so the JVM becomes PID 1 and receives `docker stop` SIGTERM. The
    `installDist` output places the launch script at `/app/bin/forwarder`.
  </action>
