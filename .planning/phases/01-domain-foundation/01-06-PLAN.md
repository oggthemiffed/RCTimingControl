**api.ts** — Axios instance with interceptors (per PATTERNS.md Axios API Client):
```typescript
import axios from 'axios';
import { getAccessToken, clearAccessToken } from './auth';

const api = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '',
  withCredentials: true,  // sends HttpOnly refresh cookie
});

api.interceptors.request.use(config => {
  const token = getAccessToken();
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

let isRefreshing = false;
let failedQueue: Array<{ resolve: (v: unknown) => void; reject: (e: unknown) => void }> = [];

api.interceptors.response.use(
  response => response,
  async error => {
    const originalRequest = error.config;
    if (error.response?.status === 401 && !originalRequest._retry) {
      if (isRefreshing) {
        return new Promise((resolve, reject) => {
          failedQueue.push({ resolve, reject });
        }).then(() => api(originalRequest));
      }
      originalRequest._retry = true;
      isRefreshing = true;
      try {
        const { data } = await axios.post('/api/v1/auth/refresh', {}, { withCredentials: true });
        const { setAccessToken } = await import('./auth');
        setAccessToken(data.accessToken);
        failedQueue.forEach(p => p.resolve(undefined));
        failedQueue = [];
        return api(originalRequest);
      } catch {
        failedQueue.forEach(p => p.reject(error));
        failedQueue = [];
        clearAccessToken();
        window.location.href = '/login';
      } finally {
        isRefreshing = false;
      }
    }
    return Promise.reject(error);
  }
);

export default api;
```

**AuthProvider.tsx** — React Context (per UI-SPEC section 6):
```typescript
interface AuthUser {
  id: string;
  email: string;
  firstName: string;
  lastName: string;
  roles: Array<'RACER' | 'ADMIN' | 'RACE_DIRECTOR' | 'REFEREE'>;
}

interface AuthContextValue {
  user: AuthUser | null;
  accessToken: string | null;
  login: (email: string, password: string) => Promise<void>;
  logout: () => void;
  isLoading: boolean;
}
```

Key behaviors:
- On mount: `POST /api/v1/auth/refresh` silently. If succeeds, decode user from response, set access token. If fails, user remains null.
- While checking: `isLoading = true` — show `<Loader2 className="animate-spin">` centered
- `login()`: POST to `/api/v1/auth/login`, store access token in memory, parse user from response, navigate to `/admin` if any staff role, else `/racer`
- `logout()`: clear access token, set user to null, navigate to `/login`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Spring Boot | All API calls go through Vite proxy in dev; CORS not configured |
| Access token storage | In-memory only; XSS cannot extract from localStorage |
| Refresh token | HttpOnly cookie; JavaScript cannot read it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-21 | Information Disclosure | Access token | mitigate | Token stored in module-level variable, never localStorage/sessionStorage; XSS attack cannot read cookies (HttpOnly) or module-scoped variables (no DOM exposure) |
| T-01-22 | Spoofing | ProtectedRoute | mitigate | Client-side guard is convenience only; backend independently enforces auth on every request via JWT filter |
| T-01-23 | Information Disclosure | Error messages | mitigate | Login shows generic "Invalid email or password" (no credential distinction); forgot-password always shows success (no email enumeration) |
</threat_model>
