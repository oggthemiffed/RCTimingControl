## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip spacing |
| sm | 8px | Compact form row spacing, list item padding |
| md | 16px | Default element spacing, card padding, form field gap |
| lg | 24px | Section padding, sidebar group gap |

## Layout Contract

### Admin Panel Layout (D-01, D-02, D-03)

**Desktop (md+):**
- Fixed left sidebar, 240px wide
- Sidebar grouped with dividers:
  - Group 1 "Events & Competitions": Events, Championships
  - Group 2 "Configuration": Tracks, Race Formats, Club Profile, Car Tag Categories
- Main content area fills remaining width with `pl-6 pr-6 pt-6` padding
- Active sidebar item: `text-primary font-medium` + 3px left border in `--primary`

**Mobile (< md):**
- Sidebar hidden; hamburger icon in top bar triggers a `<Sheet>` (slide-in drawer) from the left — use the existing `sheet.tsx` component
- Bottom fixed navigation bar (`h-14`, `fixed bottom-0 inset-x-0`) showing the 6 section icons — identical pattern to `RacerPortalLayout.tsx`
- Bottom nav items: Events, Championships, Tracks, Formats, Club, Categories

**Component to create:** `AdminPanelLayout.tsx` in `frontend/src/pages/admin/` — adapt `RacerPortalLayout.tsx` replacing the top nav links with the sidebar structure described above.

### Route Structure

```
/admin                          → redirect to /admin/events
/admin/events                   → event list page
/admin/events/:id               → event detail page (state machine, classes, entries)
/admin/championships            → championship list page
/admin/championships/:id        → championship detail + standings
/admin/tracks                   → track list + inline create/edit
/admin/formats                  → race format template list + create/edit
/admin/club                     → club profile form
/admin/categories               → car tag category management
```

Source: Claude's Discretion from CONTEXT.md.

## Interaction Contracts

### Event Status Badge + Action Buttons (D-04)

- Status badge placed in the event detail page header, inline with the event name
- Only valid next-state buttons are rendered — invalid transitions are never shown
- Action buttons are `<Button variant="default">` except Close Entries and Withdraw Entry which are `<Button variant="destructive">`
- Buttons are disabled and show a spinner icon during mutation pending state
- On successful transition: invalidate event query, show Sonner toast with new state name ("Event is now Open for entries")
- On HTTP 409: show Sonner toast "Transition rejected — page has been refreshed" and refetch event data

### Format Type-Switching Form (D-19, Specifics)

- Format type is a `<Select>` at the top of the form
- When type changes, the fields below animate out (150ms fade + translate-y) and the new field set animates in (150ms fade + translate-y) — use `tw-animate-css` classes already in index.css (`animate-in`, `fade-in`, `slide-in-from-top-2`)
- Fields that are shared across all types (name, gap between races) remain static and do not animate
- Zod schema is discriminated union matching the server's sealed interface — validation runs on submit, not on type change

### Championship Points Scale (D-13)

- Editable table with one row per finishing position (1st–20th shown by default, extendable)
- Each cell is an `<Input type="number" min="0">` rendered inline in the table
- Preset buttons above the table: "ROAR Defaults" and "BRCA Defaults" — clicking pre-populates all cells (no confirmation needed, user can still edit after)
- ROAR defaults: 1st=100, 2nd=80, 3rd=65, 4th=55, 5th=46, 6th=38, 7th=31, 8th=25, 9th=20, 10th=16, 11th=13, 12th=11, 13th=10, 14th=9, 15th=8, 16th=7, 17th=6, 18th=5, 19th=4, 20th=3
- BRCA defaults: 1st=100, 2nd=90, 3rd=83, 4th=77, 5th=72, 6th=67, 7th=62, 8th=58, 9th=54, 10th=50, 11th=47, 12th=44, 13th=41, 14th=38, 15th=35, 16th=32, 17th=29, 18th=26, 19th=23, 20th=20

Source: Claude's Discretion from CONTEXT.md.

### Championship Standings Table (D-17)

- Columns: Position, Driver Name, Total Points, then one column per round
- Drop scores: cell `opacity-40` class, no tooltip needed
- Best-X rounds highlighted: cell `font-semibold`
- DNF/DNS/DQ shown as `<Badge>` inline in cell — not as "0" — using result-type badge classes from Color section above
- Excluded drivers (CHAMP-09): row shows "EXCLUDED" in the round cell with `bg-muted text-muted-foreground` badge; admin "Exclude" action is a button in the driver row's action column

### Format Override Edit UI

- Override fields are shown as a collapsed section "Format Overrides" on the event class row (D-07)
- Clicking "Edit Overrides" expands an inline form beneath the class row showing only the fields that differ from the template snapshot
- Fields that match the template snapshot are shown as read-only with a "Reset to template" link beside them
- Fields that have been overridden show their current value with a visual indicator (amber left border `border-l-2 border-amber-400`)

Source: Claude's Discretion from CONTEXT.md.

### Logo Upload (D-22)

- `<input type="file" accept=".png,.svg">` wrapped in a styled drop zone using Tailwind `border-dashed border-2 border-border rounded-lg`
- Shows current logo preview if one exists (`<img>` with `max-h-16 object-contain`)
- File size limit: 2 MB — validated client-side before upload, error shown inline
- Upload triggers `PUT /api/admin/club/logo` — multipart form data
- On success: preview updates immediately, Sonner toast "Logo saved"
