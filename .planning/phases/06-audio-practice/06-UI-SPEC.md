## Spacing Scale

Standard 8-point scale — identical to Phases 4 and 5.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline badge padding |
| sm | 8px | Compact button padding, field group gaps |
| md | 16px | Form field spacing, dialog section gaps |
| lg | 24px | Section breaks within card/panel |
| xl | 32px | Layout column gap |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Toggle row height in audio settings panel: 40px (`h-10`) — matches shadcn Switch row pattern.
- Practice live-timing row: 48px minimum height — matches Phase 4 live timing rows.
- "Preview" audio button: 36px (`h-9`) — aligns with shadcn `size="sm"` button convention.

## Typography

Same scale as Phases 4/5. No new roles introduced.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Figtree Variable | Form labels, helper text, toggle labels in audio panel |
| Label | 12px (text-xs) | 400 (regular) | 1.4 | Figtree Variable | Section sub-labels, "phonetic spelling" hint text |
| Heading | 16px (text-base) | 600 (semibold) | 1.2 | Figtree Variable | Dialog titles, card section headings |
| Display | 20px (text-xl) | 600 (semibold) | 1.1 | Figtree Variable | Page section headings (admin audio settings page) |
| Mono | 14px (text-sm) | 400 (regular) | 1.0 | JetBrains Mono Variable | Lap times, best consecutive lap time display |

Rules:
- Best-consecutive-laps time values use `.font-mono tabular-nums` — matches live timing panel convention.
- Phonetic spelling helper text: `text-xs text-muted-foreground`.
- Two weights only: `font-normal` (400) and `font-semibold` (600). No `font-medium`.

## Color

All values are CSS custom properties from `index.css`. Do not hard-code hex or oklch literals.

| Role | CSS Variable | Approx. Value | Usage |
|------|-------------|---------------|-------|
| Dominant (60%) | `--background` | oklch(0.985 0.002 60) light / oklch(0.120 0.003 250) dark | Page background, practice session shell |
| Secondary (30%) | `--card` / `--secondary` | oklch(1 0 0) light / oklch(0.185 0.004 250) dark | Audio settings card, practice display card, admin blocklist card |
| Accent (10%) | `--primary` | oklch(0.702 0.187 41.8) — Glasgow RC orange | See reserved list below |
| Destructive | `--destructive` | oklch(0.605 0.235 27) | Blocklist remove actions only |

**Accent (`--primary`) reserved for:**
1. "Preview" button on racer profile voice section
2. "Start Practice" / "Stop Practice" primary CTA buttons
3. "Save Changes" on admin audio settings page
4. Active toggle state (shadcn Switch `checked` state — built into component)

**Destructive (`--destructive`) reserved for:**
1. "Remove" button on profanity blocklist custom terms
2. "Stop Practice" when session is RUNNING (matches "Stop Race" pattern from Phase 4)
3. Profanity validation error text on profile/phonetic name field

**Semantic color reuse (no new CSS variables):**
- `--flag-green`: audio enabled indicator dot in race control panel
- `--flag-yellow`: audio partially enabled (some toggles off) indicator
- `--flag-red`: audio disabled indicator / clip cache error state
- `--chart-1` (orange): improvement lap beep visual flash on live timing row (AUDIO-04)
- `--chart-2` (blue): non-improvement lap beep visual flash on live timing row (AUDIO-04)

## Surface Specifications

### Surface 1 — Racer Profile: Voice Section

**Location:** `frontend/src/pages/racer/ProfilePage.tsx`

**Placement:** New `<Card>` section added **below** the existing profile form card (after the membership/affiliation card if present), before the page footer.

**Note:** `phoneticName` field already exists in the ProfilePage form schema and form fields. The voice section adds voice selector + preview button as a **separate card** from the main profile form to avoid coupling the save actions.

**Card title:** "Announcement Voice"

**Card layout:**
```
┌─ Card ───────────────────────────────────────────┐
│  Announcement Voice                               │
│  ─────────────────────────────────────────────── │
│                                                   │
│  Phonetic name                                    │
│  ┌──────────────────────────────────────────────┐│
│  │  e.g. "Tom-is JONES"                         ││
│  └──────────────────────────────────────────────┘│
│  ⓘ How your name sounds when announced. Leave    │
│     empty to use your display name.              │
│                                                   │
│  Voice                                            │
│  ┌──────────────────────────────────────────────┐│
│  │  Alan (en-GB) — club default            ▼   ││
│  └──────────────────────────────────────────────┘│
│                                                   │
│  ▶  Preview  [Save Voice Preferences]             │
└──────────────────────────────────────────────────┘
```

**Field details:**

- **Phonetic spelling input:** `<Input placeholder='e.g. "Tom-is JONES"' />` with `<FormMessage />` shown on profanity rejection. Maps to existing `phoneticName` schema field. Helper text: "How your name sounds when announced. Leave empty to use your display name."
- **Voice selector:** `<Select>` populated from `GET /api/v1/audio/voices` — list of `{ voiceId: string, label: string, isDefault: boolean }`. Default option label: `"{voiceName} — club default"`. Non-default options label: `"{voiceName}"`.
- **Preview button:** `<Button variant="outline" size="sm">` with `<Play className="mr-2 h-4 w-4" />` icon. Label: "Preview". On click: calls `GET /api/racer/me/name-clip?voice={selectedVoiceId}` (or generates on demand). Plays returned `.wav` via `new Audio(url).play()`. While loading: button shows `<Loader2 className="animate-spin" />`, disabled. After play completes: returns to normal state.
- **Save Voice Preferences button:** `<Button variant="default">` (primary/orange). Saves `voiceId` + `phoneticName` together via `PATCH /api/racer/me/profile`. Shows loading spinner, then Sonner toast on success/error.

**Validation:**
- `phoneticName` blocked profanity: show `<FormMessage>` "This name contains inappropriate content and cannot be saved." (AUDIO-14)
- Preview failure: Sonner toast (error variant) "Could not generate preview. Please try again."

**States:**
| State | Visual |
|-------|--------|
| Loading voices | Select shows "Loading voices…" disabled |
| Voice list loaded | Select shows available voices |
| No voices available | Select shows "No voices configured" disabled; Preview hidden |

### Surface 3 — Race Control Cockpit: Audio Settings Panel

**Location:** `frontend/src/pages/race-control/panels/AudioSettingsPanel.tsx`, inserted in `CockpitPage.tsx`.

**Placement:** Collapsible panel — added to the right panel area as a **collapsible section** above the existing race-state-dependent content. Collapsed by default; expanded by clicking "Audio Settings" header row.

**Trigger row layout:** `flex h-10 items-center gap-2 px-4 border-b cursor-pointer select-none` — clicking anywhere on the row toggles open/close.

```
[Audio icon (Volume2, h-4 w-4)]  Audio Settings  [status dot]  [ChevronDown/Up]
```

Status dot indicates overall audio state:
- All enabled: `--flag-green` dot, `aria-label="Audio: all enabled"`
- Some disabled: `--flag-yellow` dot, `aria-label="Audio: partially enabled"`
- All disabled: `--flag-red` dot, `aria-label="Audio: disabled"`

**Expanded panel layout:**
```
┌─ Collapsible ─────────────────────────────────────────────────────┐
│  [Section heading: "Announcement Types" — text-xs semibold muted] │
│  ┌─ Toggle rows ──────────────────────────────────────────────┐   │
│  │  Countdown intervals         [Switch]                      │   │
│  │  Stagger car calls           [Switch]                      │   │
│  │  Lap improvement beeps       [Switch]                      │   │
│  │  Finish announcements        [Switch]                      │   │
│  │  Running order               [Switch]                      │   │
│  └────────────────────────────────────────────────────────────┘   │
│                                                                    │
│  Running order depth                                               │
│  ┌────────────┐                                                    │
│  │     3      │  positions announced                              │
│  └────────────┘                                                    │
│                                                                    │
│  [Section heading: "Volume" — text-xs semibold muted]             │
│  [Slider: 0–100, default 80, step 5]   [Volume2/VolumeX icon]    │
│                                                                    │
│  [Button variant="outline" size="sm": ▶ Test Audio]               │
│  [text-xs muted: "Plays a sample announcement to test speakers"]  │
└────────────────────────────────────────────────────────────────────┘
```

**Toggle rows — anatomy:**
Each row: `flex items-center justify-between h-10 px-4`
- Left: label (`text-sm font-normal`)
- Right: shadcn `Switch` component — `checked` state wired to per-type boolean; fires mutation on change

**Running order depth field:**
`<Input type="number" min="1" max="20" className="w-20 h-8 text-center" />` with "positions announced" label inline. Mutation fires on `onBlur`. Value range 1–20; invalid value reverts to last saved value.

**Volume slider:**
`<Slider min={0} max={100} step={5} />`. Controls `HTMLAudioElement.volume` on the client audio engine. No server mutation — local preference stored in `localStorage` under key `rc-audio-volume`. Mute/unmute icon toggles at left of slider: `Volume2` when volume > 0, `VolumeX` when 0. Clicking icon toggles mute (stores last non-zero value).

**Test Audio button:**
`variant="outline" size="sm"`. Plays a short sample clip via Web Speech API (`"Test. Race one. Positions: first, Car 12. Second, Car 7. Third, Car 3."`). No server call. Verifies browser TTS output is audible. While playing: button shows `<Loader2 />`, disabled.

**Data source:** `GET /api/v1/race-control/settings/audio` — returns `AudioSettingsDto`. Mutations: `PATCH /api/v1/race-control/settings/audio`. Settings persist per club/session (admin-configured defaults; overridable per meeting from this panel).

### Surface 7 — Practice Live Table (`PracticeLiveTable`)

**Location:** `frontend/src/pages/race-control/panels/PracticeLiveTable.tsx`

**Data source:** STOMP topic `/topic/practice/{sessionId}/timing`. Follows `useLiveTiming.ts` pattern. Each message is a `PracticeTimingDto[]` replacing the full table state.

**Column specification:**

| # | Column | Width | Typography | Notes |
|---|--------|-------|------------|-------|
| 1 | Pos | w-12 | 20px semibold | Auto-calculated; `—` if < 1 lap |
| 2 | Racer | flex-1 | 14px body | Display name; truncate |
| 3 | Laps | w-16 | 14px tabular-nums mono | Lap count |
| 4 | Best Lap | w-28 | 14px mono tabular-nums | Format: `m:ss.sss`; `—` if no laps |
| 5 | Best {N} Consec. | w-32 | 14px mono tabular-nums | Format: `m:ss.sss avg`; `—` if < N laps complete |
| 6 | Last Lap | w-28 | 14px mono tabular-nums | Last crossing time |
| 7 | Status | w-24 | text-xs | Badge: "Known" (outline) / "Linked" (secondary) / "Unknown" (destructive outline) |

**"Best {N} Consec." column header:** Dynamic — reads N from session config. Renders as `"Best {N} Laps"`. Tooltip on hover: "Best average lap time over any {N} consecutive laps."

**Row improvement flash (AUDIO-04 visual correlation):**
- When a new passing arrives improving the racer's best lap: row briefly flashes `bg-[var(--chart-1)]/20` (orange tint) for 800ms via CSS `animate-flash` keyframe.
- Non-improving passing: row briefly flashes `bg-[var(--chart-2)]/15` (blue tint) for 400ms.

**Unknown transponder rows:**
- Shown in table with racer column = `font-mono text-xs text-muted-foreground` — raw transponder number.
- Status badge: `variant="outline" className="border-destructive text-destructive"` — "Unknown".
- Row background: `bg-[var(--flag-red)]/5` — subtle red tint to surface unknown entries visually.

**Table sort:** Default sort by laps desc, then best lap asc. Not user-sortable during RUNNING state. User-sortable (column headers) when STOPPED.

**Empty state (no passings yet):**
Full-width row spanning all columns: "Waiting for transponders…" (`text-sm text-muted-foreground text-center py-8`)

## Accessibility Notes

- All `Switch` components in audio settings: each has an associated `<label>` with `htmlFor` pointing to the switch `id`. Label text is the announcement type name.
- "Preview" button: `aria-label="Preview name announcement"`. Loading state: `aria-busy="true"`.
- Practice live table: `role="table"` with `aria-label="Live practice timing"`. Unknown transponder rows: `aria-label="Unknown transponder {id}"`.
- Practice session state badge: `aria-live="polite"` — announces state changes to screen readers.
- Volume slider: `aria-label="Announcement volume"`, `aria-valuetext="{value}%"`.
- "Stop Practice" destructive button: `aria-label="Stop practice session"`. No confirmation dialog — irreversibility is low risk; the session can be restarted.
- Profanity blocklist "Remove" buttons: `aria-label="Remove term {term} from blocklist"`.
- Unknown transponder alert banner: `role="alert"` — announces when new unknown transponders appear during a session.
