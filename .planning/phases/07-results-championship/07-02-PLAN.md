**RoundGeneratorService — qualifying car_number assignment:**

In the inner class `persist()` method, when creating qualifying RaceEntry rows, assign `carNumber` as a 1-based counter per race. Each race's entries are numbered independently: car 1 is the entry with `gridPosition = 1`, car 2 is `gridPosition = 2`, etc. The assignment must happen at the point where `re.setGridPosition(gridPosition)` is called. Add immediately after:

```java
re.setCarNumber(gridPosition);  // car_number = 1-N, same as grid_position for qualifying heats
```

(Grid position in qualifying is assigned 1..N in the order entries are slotted into heats. Car number equals grid position for qualifying — RESEARCH.md Pattern 4 / Assumption A4.)

**BumpUpSeedingService — finals car_number assignment:**

In `seedFinals()`, the `qualifyingStandings` parameter is a `List<Long>` of entryIds in qualifying standing order (position 1 first). When creating final RaceEntry rows, assign car_number = the driver's qualifying standing position. The index in the `qualifyingStandings` list is 0-based, so:

```java
int carNum = qualifyingStandings.indexOf(entryId) + 1;
// If entryId not found in qualifyingStandings (e.g., bump-up driver), assign car_number = null
entry.setCarNumber(carNum > 0 ? carNum : null);
```

For bump slots (`bumped = true`), set carNumber to null — bump-up drivers receive a number only after they actually bump up via `applyBumpUpResults()`. In `applyBumpUpResults()`, the bumpSlot entries that get an actual entryId assigned should also get a carNumber. Read the method to determine how to set it; use the position among bump slots (bumpSlots.indexOf(slot) + existingNonBumpCount + 1) or null if too complex — null is acceptable per Assumption A4.
