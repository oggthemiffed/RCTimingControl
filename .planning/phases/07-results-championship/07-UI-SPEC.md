## Spacing Scale

Declared values (multiples of 4 only). Standard 8-point scale applies throughout.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline spacing within table cells |
| sm | 8px | Compact element spacing, table row padding |
| md | 16px | Default section/card internal padding |
| lg | 24px | Section padding, card gaps |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom padding |

**Exceptions:**
- Touch targets on mobile expandable rows: minimum 44px tap height (WCAG 2.5.5). Use `min-h-[44px]` on the expandable trigger row.
- Print layout: use `p-4` (`16px`) not `p-8` to conserve paper space. Apply via `print:p-4` as established in `PrintResultsPage.tsx`.
- Expandable lap-time rows: `py-1` (`4px`) per lap row — high-density data display.

## Typography

All sizes use CSS custom property `--font-sans` (Figtree Variable) for UI, `--font-mono` (JetBrains Mono Variable) for all numeric timing data. Existing `.laptime`, `.timing`, `code`, `.mono` utility classes apply monospace with `tabular-nums` — use these on every lap time cell.

| Role | Size (Tailwind) | px | Weight | Line Height | Usage |
|------|-----------------|----|--------|-------------|-------|
| Body | `text-sm` | 14px | 400 (regular) | 1.5 | Table cell content, metadata, secondary labels |
| Label | `text-base` | 16px | 400 (regular) | 1.5 | Form labels, nav items, card body text |
| Display | `text-xl` | 20px | 700 (bold) | 1.3 | Section headings, championship name on public page |
| Heading | `text-2xl` | 24px | 700 (bold) | 1.2 | Page title, race label in results header |

**Two weights only: 400 (regular) and 700 (bold).** Do not introduce weight 600 (semibold).

**Mono — timing data:**
- All lap times, total times, gaps, and position numbers in results tables: use `font-mono tabular-nums` (or Tailwind `font-mono`).
- Point scores in championship standings: use `font-mono` to maintain column alignment.

**Existing pattern from `PrintResultsPage.tsx` — preserve:**
- Page heading: `text-2xl font-bold`
- Table content: `text-sm`
- Monospace numeric cells: `font-mono`

All values are CSS custom properties already defined in `index.css`. Do not introduce new hex literals; reference tokens.

| Role | CSS Token | Light value | Usage |
|------|-----------|-------------|-------|
| Dominant (60%) | `--background` | `oklch(0.985 0.002 60)` — warm taupe | Page background, print surface |
| Secondary (30%) | `--card` / `--muted` | `oklch(1 0 0)` / `oklch(0.960 0.003 50)` | Result cards, standings container, expandable lap section background |
| Accent (10%) | `--primary` | `oklch(0.702 0.187 41.8)` — Glasgow RC orange | Reserved list below |
| Destructive | `--destructive` | `oklch(0.605 0.235 27)` | No destructive actions in this phase (read-only public pages) |

**Accent (`--primary`) reserved for:**
1. "Print" button (matches existing `PrintResultsPage` button)
2. Active nav item indicator in racer portal (matches existing `border-primary` pattern)
3. Results link and Standings link on event schedule — these are calls-to-action, not just navigation
4. Position 1 (P1) marker in results table — use `text-primary font-bold` on first-place row

**Race-domain semantic colors (from `index.css`) — use where appropriate:**
- `--lap-personal-best` (purple): highlight a racer's personal best lap in the expandable lap times list
- `--lap-absolute-best` (green): highlight overall session best lap in the expandable lap times list
- `--flag-red` / `--destructive`: DNS / DQ / EXC badge backgrounds in standings (matches existing `ChampionshipStandingsTable` pattern)

**Drop score visual treatment (CONTEXT.md D-10, Claude's Discretion):**
Use `text-muted-foreground line-through` on dropped round cells. This is already implemented in `ChampionshipStandingsTable.tsx` (`result.dropped` branch uses `text-slate-300 line-through`). Replace `text-slate-300` with `text-muted-foreground` to use the token correctly in both light and dark modes.

## Interaction Contract

### Expandable lap times row (CONTEXT.md D-02, Specifics)

- Trigger: entire result row is tappable — no separate expand icon required, but include a chevron (`ri-arrow-down-s-line`) on the right side to communicate affordance.
- State: closed by default. One racer expanded at a time (accordion behaviour — opening a new row closes the previous).
- Expanded content: a sub-row spanning the full table width, containing a compact lap-time list (`font-mono text-xs`) with lap number, lap time, and personal-best / session-best highlight.
- Mobile: expanded row must have `min-h-[44px]` on the trigger row. Touch tap to expand works without hover dependency.
- Implementation hint: use shadcn `Collapsible` (Radix UI Collapsible primitive) for each row to get accessible keyboard expand/collapse. Do not implement with raw `<details>` (no animation control).

### Racer history list (CONTEXT.md D-05)

- Top-level: list of events the racer entered. Each event card is expandable to show race results.
- Same accordion pattern as lap times: chevron affordance, one-at-a-time expansion.
- Each race result row links to `/results/:raceId` via a `ri-external-link-line` icon at row end.
- Empty state: "No results yet. Results appear here after your first race finishes."

### Public standings page (CONTEXT.md D-09, D-10)

- Clean, no admin controls. Championship name as `text-2xl font-bold`, description as `text-sm text-muted-foreground`.
- Reuse `ChampionshipStandingsTable` component but correct the drop-score token (`text-muted-foreground line-through` not `text-slate-300 line-through`).
- No pagination required — small club, full table on one page.

### Event schedule results/standings links (CONTEXT.md D-03, D-11)

- "View Results" link appears beside a finished race entry when race state = FINISHED.
- "View Standings" link appears on an event card when the event is associated with a championship.
- Both render as `<Button variant="outline" size="sm">` with a leading icon (`ri-trophy-line` for standings, `ri-file-list-line` for results).

### Car tag display in printed results (CONTEXT.md D-07, D-08)

- When club setting `showCarTagsInResults = true`: display car tags as small key/value pairs beneath the driver name in the results table. Use `text-xs text-muted-foreground` and a comma-separated inline format: `Chassis: Xray T4 · ESC: Hobbywing 60A`.
- When `showCarTagsInResults = false`: omit entirely, do not show an empty space.
- Admin toggle location: Admin → Club settings page (existing club profile page). Render as a `Switch` labelled "Show car details in printed results".

## Page-Level Layout Specifications

### `/results/:raceId` — Public Results Page

Built on existing `PrintResultsPage` component mounted on a public (unprotected) route.

```
┌─────────────────────────────────────────────────┐
│ [Club Logo]      Race label           [Date/time]│  ← header, max-w-3xl mx-auto
├─────────────────────────────────────────────────┤
│ Pos │ Driver          │ Car# │ Laps │ Time │ Best│ ← results table
│  1  │ Jane Smith      │  3   │  14  │ 5:12 │0:21 │   expandable rows
│     ↳ LAP TIMES: 0:22 · 0:21 · 0:23 ...        │   (collapsed by default)
│  2  │ Alan Jones      │  7   │  14  │ 5:14 │0:21 │
├─────────────────────────────────────────────────┤
│ [Print Results]                    print:hidden  │
└─────────────────────────────────────────────────┘
```

- Container: `max-w-3xl mx-auto p-8 print:p-4`
- No login required. No navigation chrome.
- Print stylesheet hides the Print button: `print:hidden` (already implemented)

### `/championships/:id` — Public Championship Page

Separate from admin `ChampionshipDetailPage`.

```
┌─────────────────────────────────────────────────┐
│  Championship Name                               │  ← text-2xl font-bold
│  Season description / scoring rule               │  ← text-sm text-muted-foreground
├─────────────────────────────────────────────────┤
│  # │ Driver       │ Total │ R1 │ R2 │ R3 │ R4   │  ← ChampionshipStandingsTable
│  1 │ Jane Smith   │  42   │ 15 │ 12 │ ~~10~~ │15│     dropped = line-through muted
│  2 │ Alan Jones   │  38   │ 12 │ 15 │ 11    │...│
└─────────────────────────────────────────────────┘
```

- Container: `max-w-4xl mx-auto p-8`
- No login required. No admin controls. No edit buttons.

### Racer Portal — Results Tab

New tab added to `RacerPortalLayout` alongside Profile, Cars, Transponders, Entries.

Nav item: `{ to: '/racer/results', label: 'Results', Icon: Trophy }` — use `ri-trophy-line` from remixicon.

```
┌─────────────────────────────────────────────────┐
│  Results                                         │  ← page heading
├─────────────────────────────────────────────────┤
│  ▼ Round 1 — Club Championship 2026              │  ← expandable event card
│    Class A Final  │ P2 │ 14 laps │ 5:14  [→]    │     links to /results/:raceId
│    Class A Q1     │ P1 │ 8 laps  │ 2:30  [→]    │
├─────────────────────────────────────────────────┤
│  ▶ Round 2 — Club Championship 2026  (collapsed) │
└─────────────────────────────────────────────────┘
```

- Use shadcn `Collapsible` for each event card.
- Event cards sorted most recent first.
