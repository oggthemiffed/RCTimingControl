    Public API:
      - `public List<RoundPreviewDto> preview(RoundGenerationRequest request)` — computes what would be generated without persisting; returns preview rows ordered by sequenceInEvent.
      - `public void generate(RoundGenerationRequest request)` — persists all Round + Race + RaceEntry records in a single `@Transactional` call. Throws `IllegalStateException` with message "Run order already generated for event {id}" if `roundRepository.existsByEventId(request.eventId())`.

    Algorithm (both `preview` and `generate` share via a private helper):
      1. Load event → list of `EventClass` for that event → for each class load `Entry` rows with `CONFIRMED` lifecycle.
      2. For each class: compute heat count = `ceil(entries.size() / maxCarsPerHeat)`. If entries.size ≤ maxCarsPerHeat, heat count = 1.
      3. Split drivers into heats by seeding them in `abilityRating DESC` order (from UserClassRating for this class) using **snake draft**: heat 1 takes seed 1, heat 2 seed 2, ..., heat N seed N, then reverse — heat N seed N+1, ..., heat 1 seed 2N, then forward again. Drivers without a rating get default 50 and sort last (tie-break by entryId asc).
      4. Heat assignment is **fixed across all rounds** (same drivers in Heat 1 across Practice 1..N and Qualifying 1..M) per HEAT-STRUCTURE-SPEC.
      5. Build Round records in order: Practice 1, Practice 2, ..., Practice N, Qualifying 1, Qualifying 2, ..., Qualifying M, then Finals (finals grids are created empty here; seeding happens later via `BumpUpSeedingService` when qualifying completes). Assign `sequenceInEvent` contiguously starting at 1.
      6. For each Round, for each EventClass, create one Race per heat: `startType = STAGGER` for PRACTICE/QUALIFIER, `startType = GRID` for FINAL. `finalLetter = null` for practice/qualifier, `"A"`, `"B"`, `"C"` ... for finals (char 'A' + finalIndex).
      7. For Round 1 (first Practice or first Qualifier if no practice), set `RaceEntry.gridPosition = seed order within heat` (1..N).
      8. For subsequent Round races, set `gridPosition = null` — it will be assigned at the moment the previous round finishes via the state machine transition in plan 05 (which will call a method `applyPreviousRoundFinishingOrder(raceId)`). Document this with a comment.
      9. For FINAL races, leave all `RaceEntry` with `gridPosition = null` — `BumpUpSeedingService` fills them after qualifying completes.

    Include a helper `public void applyPreviousRoundFinishingOrder(Long newRaceId, List<Long> entryIdsInFinishingOrder)` that sets `gridPosition` on `RaceEntry` rows for a specific race using finishing order from the previous round's same heat — best-finisher (index 0) goes to `gridPosition = 1` per HEAT-STRUCTURE-SPEC §"Start Order Rules".

    Handle missing finals config: if `EventClass.finalsCount` is null, look up the per-class override in `request.classFinalsConfigs()`; if still null, default to `finalsCount=1, carsPerFinal=maxCarsPerHeat, bumpCount=0`.
