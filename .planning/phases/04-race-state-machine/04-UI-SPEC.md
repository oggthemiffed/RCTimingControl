## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, chip inner padding |
| sm | 8px | Row inner padding, badge padding |
| md | 16px | Panel inner padding, form field spacing |
| lg | 24px | Panel header padding, section separation |
| xl | 32px | Layout column gap |
| 2xl | 48px | Major section breaks within right panel |
| 3xl | 64px | (reserved — not used at cockpit scale) |

Exceptions:
- Touch targets for +1/−1 marshal buttons and state-transition buttons: minimum 44px height (accessibility requirement for trackside mobile use per D-03).
- Run order list item row height: 48px minimum to be tappable on mobile.
- Bottom nav bar height: 56px (h-14, matches RacerPortalLayout and AdminPanelLayout).

## Typography

All sizes use Figtree Variable, matching the existing system.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 14px | 600 (semibold) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.1 |

Notes:
- Live timing table uses 14px body weight 400 for lap times and gaps; position number uses 20px semibold.
- Race state badge text uses 12px semibold (existing Badge component xs height of 20px).
- Missed-marshal count emphasis ("missed 2 this event") uses 14px semibold destructive color.
- Timer/clock display (countdown before race start, race elapsed time) uses 28px display semibold, tabular-nums variant for non-jittery rendering.

Source: Prior phases use same scale; confirmed consistent with `index.css` font-sans assignment.

## Color

Tokens map directly to the CSS custom properties already declared in `frontend/src/index.css`.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (oklch 1 0 0 / dark oklch 0.147) | Page background, cockpit chrome |
| Secondary (30%) | `--card` / `--sidebar` | Left run-order panel background, right panel surface, pre-race readiness columns |
| Accent (10%) | `--primary` (oklch 0.5 0.134 242 — blue) | See reserved list below |
| Destructive | `--destructive` (oklch 0.577 0.245 27 — red) | See reserved list below |

Accent (`--primary`) reserved for:
1. Active/selected race row highlight in run order list (left panel)
2. Primary state-transition buttons: "Call Grid", "Start Race" (contained, `variant="default"`)
3. Active nav item indicator (border-l-[3px] border-primary, matching AdminPanelLayout pattern)
4. STOMP connection status indicator when connected (green dot — use `chart-1` oklch 0.879)

Destructive (`--destructive`) reserved for:
1. "Abandon Race" button
2. Marshal adjustment: −1 lap button
3. Missed-marshal count when count ≥ 2 ("missed N this event")
4. Proximity alert row highlight in referee view
5. Race state badge when race is STOPPED (badge variant `destructive`)

Additional semantic colors (from existing chart palette, read-only — do not introduce new CSS variables):
- `--chart-1` (oklch 0.879 / yellow-green): STOMP connected indicator, improvement lap beep (Phase 6)
- `--chart-3` (oklch 0.666 / orange): backmarker warning badge in referee view
- `--muted-foreground`: secondary labels, timestamps in audit trail, gap-to-leader column

Race state badge color mapping (use Badge component):
| State | Badge variant | Semantic |
|-------|--------------|----------|
| PENDING | outline | Neutral |
| GRID | secondary | Neutral active |
| RUNNING | default (primary) | Active / in progress |
| STOPPED | destructive | Warning |
| FINISHED | outline | Done |

Source: `frontend/src/index.css` CSS custom properties, `frontend/src/components/ui/badge.tsx` variants.

## Layout Contract

### `/race-control` — Race Director Cockpit

Structure (D-01, D-02):
```
[Cockpit — full viewport height, no scroll on outer container]
  [Top bar — h-14, border-b, contains: "Race Control" title + event name + user + logout]
  [Body — flex row, fills remaining height]
    [Left panel — w-[30%] min-w-56, border-r, overflow-y-auto, bg-card]
      [Run order list OR round-generator prompt (D-12)]
    [Right panel — flex-1, overflow-y-auto]
      [State-dependent content (D-10)]
```

Mobile (D-03): At breakpoints below `md`, the two-column layout collapses to a single column. The left panel becomes a full-width drawer opened via a "Run Order" button in the top bar (Sheet component, side="left"). The right panel fills the full screen. Bottom nav bar is NOT used on the cockpit — the cockpit is a dedicated single-page tool with its own top bar only. This differs from the racer portal: cockpit has no page-to-page navigation.

Left panel — run order list item anatomy (one row per race):
```
[48px row, px-3, flex items-center gap-2]
  [Race sequence number — 20px, semibold, w-6, text-muted-foreground]
  [Race label — "Q1 Heat 2 · Touring" — 14px body, truncate, flex-1]
  [State badge — Badge component, xs]
```

Active/selected row: `bg-primary/10 border-l-[3px] border-primary` (matches AdminPanelLayout NavItem active style).

Left panel — round generator prompt (shown when no run order exists, D-12):
```
[Card, m-3, p-4]
  [Heading: "Event setup incomplete"]
  [Body: "Generate the run order before the meeting starts."]
  [Button variant="default": "Generate Run Order →"]
```

Right panel — state-dependent content:

**State: PENDING — Grid Editor (D-10, D-09)**
```
[Section header: "Grid — [Race Label]" + Badge state]
[Table: shadcn Table component]
  [Columns: Grid pos (input, w-16) | Driver name | Transponder | Car]
  [Row count = RaceEntry count]
[Footer: Button "Call Grid" variant="default" — advances to GRID state]
```

Grid position input: `<Input type="number" min="1" className="w-14 h-8 text-center" />`. On blur/enter the table row reorders immediately (optimistic local sort, mutation fires on blur).

**State: GRID — Pre-Race Readiness (D-20)**
```
[Section header: "Grid Called — [Race Label]" + Badge]
[Two-column grid, gap-4]
  [Left column: "Marshal Duty"]
    [Table: Driver | Absences this event (D-21) | checkbox "Mark absent"]
  [Right column: "Grid Call"]
    [Ordered list: grid position + driver name + car number]
[Footer: Button "Start Race" variant="default" | Button "Back to Pending" variant="outline"]
```
Marshal absence checkbox: checking it fires a mutation to record the absence. An optional "Apply Penalty" button appears per row after marking absent (separate action per D-22).

**State: RUNNING — Live Timing (D-14, D-16)**
```
[Section header: "Running — [Race Label]" + elapsed time display (28px semibold tabular-nums)]
[Live timing table]
  [Columns: Pos | Driver | Laps | Last Lap | Best Lap | Gap | +1 | −1]
  [Pos: 20px semibold]
  [+1/−1: Button variant="outline" size="icon", h-11 min (44px touch target)]
[Footer row: Button "Stop Race" variant="outline" | Button "Abandon Race" variant="destructive"]
[Collapsible "Marshal Adjustments" section (D-16)]
  [CollapsibleTrigger: "Marshal Adjustments (N)" — 14px semibold, chevron icon]
  [CollapsibleContent: Table: Time | Driver | Adjustment | Acting user]
[Synthetic timing button (dev profile only, D-07)]
  [Button variant="outline" size="sm": "Synthetic Lap" — rendered only when VITE_SPRING_PROFILE=dev]
```

**State: STOPPED**
```
[Section header: "Stopped — [Race Label]" + Badge destructive]
[Same live timing table as RUNNING, no +1/−1 buttons]
[Footer: Button "Resume Race" variant="default" | Button "Abandon Race" variant="destructive" | Button "Finish Race" variant="outline"]
[Marshal Adjustments collapsible — same as RUNNING]
```

**State: FINISHED — Results + Audit (D-19, D-23)**
```
[Section header: "Finished — [Race Label]" + Badge outline]
[Results table]
  [Columns: Pos | Driver | Laps | Total Time | Best Lap | Gap to Leader]
[Footer: Button "Print Results" variant="outline" (opens new tab, D-19)]
[Marshal Adjustments collapsible — same as RUNNING]
```

**Between races — Pre-Race Readiness (D-20)**
Displayed in the right panel after a race reaches FINISHED and before the next race is started (auto-advance selects the next race, which will be in PENDING state — so this is effectively the GRID state of the FINISHED→next-PENDING transition). See GRID state above.

### Round Generator Wizard (D-11, D-12, D-13)

Implemented as a multi-step flow within the right panel. No modal or full-page route — stays in the cockpit.

```
[Wizard container — Card, fills right panel]
  [Step indicator: "Step N of 3" — 14px muted-foreground]
  [Step 1: Practice & Qualifying config]
    [Form fields: Practice rounds (Input number), Qualifying rounds (Input number), Max cars per heat (Input number)]
    [Footer: Button "Next" variant="default"]
  [Step 2: Finals config per class]
    [Table: Class | Finals count | Cars per final | Bump spots]
    [Each cell is an inline Input, pre-populated from EventClass settings]
    [Footer: Button "Back" variant="ghost" | Button "Preview" variant="default"]
  [Step 3: Full preview]
    [Table: Sequence | Type | Round | Class | Heat | Drivers (comma list)]
    [Footer: Button "Back" variant="ghost" | Button "Generate Run Order" variant="default"]
```

All wizard inputs use React Hook Form v7 + Zod. Validation errors appear as inline field messages (shadcn Form component pattern, matching admin panel forms from Phase 3).

### `/race-control/referee` — Referee View (D-17, D-18)

Layout: single-column, full-width page. Uses the same top bar pattern as the cockpit (RaceControlLayout shared component), no left panel.

```
[Top bar — h-14: "Race Steward View" + event + race label + Badge state + user + logout]
[Main content — px-6 py-4, two-column on lg+, single-column below]
  [Left ~60%: Live timing table]
    [Same columns as cockpit RUNNING: Pos | Driver | Laps | Last Lap | Best Lap | Gap]
    [Proximity alert rows: bg-chart-3/20 highlight (orange tint) when car is closing (OFFICIAL-01)]
    [Backmarker rows: badge "LAPPED" variant="outline" with chart-3 border (OFFICIAL-02)]
  [Right ~40%: Incident & Penalty panel]
    [Section: "Incidents"]
      [Button "Raise Incident" variant="outline" size="sm" — opens Dialog]
    [Section: "Penalties"]
      [Button "Apply Penalty" variant="outline" size="sm" — opens Dialog]
    [List of incidents raised this race: driver name + description + timestamp]
    [List of penalties applied: driver name + type + value + timestamp]
```

Incident report dialog (OFFICIAL-03):
```
[Dialog: "Raise Incident"]
  [Select: Driver involved]
  [Select: Incident type (Contact / Track limits / Other)]
  [Textarea: Description]
  [Footer: Button "Cancel" variant="ghost" | Button "Raise Incident" variant="default"]
```

Penalty application dialog (OFFICIAL-04):
```
[Dialog: "Apply Penalty"]
  [Select: Driver]
  [RadioGroup: Penalty type — Lap penalty | Time penalty]
  [Input: Value (number of laps or seconds)]
  [Textarea: Reason (optional)]
  [Footer: Button "Cancel" variant="ghost" | Button "Apply Penalty" variant="destructive"]
```

## Interaction Contracts

### State Transition Confirmations

All dialogs use the shadcn `Dialog` component. Copy is prescriptive — executors use this exact wording.

| Transition | Trigger | Dialog copy | Primary button |
|-----------|---------|-------------|----------------|
| PENDING → GRID | "Call Grid" | "Call grid for [Race label]? This will announce the grid to drivers." | "Call Grid" (default) |
| GRID → RUNNING | "Start Race" | "Start [Race label]? The race clock will begin immediately." | "Start Race" (default) |
| RUNNING → STOPPED | "Stop Race" | "Stop the race? You can resume or abandon it afterwards." | "Stop Race" (outline) |
| STOPPED → RUNNING | "Resume Race" | "Resume [Race label]? The race clock will continue from where it stopped." | "Resume Race" (default) |
| RUNNING / STOPPED → FINISHED via abandon | "Abandon Race" | "Abandon [Race label]? Results will be saved up to this point and cannot be undone." | "Abandon Race" (destructive) |
| Skip forward in run order (D-05) | Clicking non-current race row | "Skip to [Race label]? This will skip [N] race(s). Confirm to override auto-advance." | "Skip" (outline) |

HTTP 409 responses (conflicting command from another session, CTRL-05): display a non-blocking toast via `sonner` — "Command rejected: another session already changed this race's state. Refresh to sync." No retry.

### Marshal Lap Confirmation (D-14)

+1 button opens Dialog:
```
Title: "Add marshal lap"
Body: "Add 1 lap to [Driver Name]? This will be recorded in the audit trail."
Cancel: "Cancel" (ghost)
Confirm: "Add Lap" (default)
```

−1 button opens Dialog:
```
Title: "Remove marshal lap"
Body: "Remove 1 lap from [Driver Name]? This will be recorded in the audit trail."
Cancel: "Cancel" (ghost)
Confirm: "Remove Lap" (destructive)
```

### Unknown Transponder Linking (CTRL-06)

When an unregistered transponder fires during a race, a banner appears above the live timing table:
```
[Banner: bg-muted, border-l-4 border-chart-3, px-4 py-3, flex items-center gap-3]
  [Icon: AlertTriangle (lucide), text-chart-3]
  [Text: "Unknown transponder [XXXXX] — not linked to any entry in this race."]
  [Button "Link to Entry" variant="outline" size="sm"]
```

"Link to Entry" opens a Dialog with a Select for the entry to link it to. On confirm, fires the link mutation and removes the banner.

### Print Results (D-19)

"Print Results" button calls `window.open('/race-control/results/{raceId}/print', '_blank')`. The print page is a separate route rendered outside the cockpit layout — white background, print-optimised CSS (`@media print`). No confirmation dialog.

Print page content (D-23):
- Club logo at top-right if configured (from club profile; rendered as `<img>`)
- Race title (class name + round + heat + final letter if applicable)
- Results table: Pos | Car # | Driver | Laps | Total Time | Best Lap | Gap
- Position chart: driver positions plotted by lap number (Chart.js `<Line>` chart, static render on page load from the result snapshot — server-side data only, no live WebSocket on print page)
- Footer: "Printed [date] — RC Timing"

### Synthetic Timing Button (D-07)

Visible only when `import.meta.env.VITE_DEV_PROFILE === 'true'`. Renders as:
```
[Button variant="outline" size="sm", positioned bottom-right of right panel]
  "Synthetic Lap"
```
Each click fires `POST /api/v1/dev/race/{raceId}/synthetic-passing`. No confirmation dialog. The button does not appear in production builds.

### STOMP Connection Status

A small indicator in the cockpit top bar, right side (before user name):
```
[span, flex items-center gap-1.5, text-xs text-muted-foreground]
  [Circle dot: 8px, bg-chart-1 if connected, bg-destructive if disconnected]
  "Timing connected" / "Timing disconnected"
```

## Accessibility Notes

- All confirmation dialogs trap focus and restore on close (Dialog component from Radix handles this).
- +1/−1 marshal buttons: `aria-label="Add marshal lap for [Driver Name]"` / `aria-label="Remove marshal lap for [Driver Name]"`. Button height minimum 44px.
- Run order list rows that are clickable (D-05 skip): `role="button"` or rendered as `<button>` — not `<div onClick>`.
- Race state badge: `aria-label="Race state: [STATE]"` to prevent screen reader reading raw badge text only.
- Live timing table updates via STOMP: use `aria-live="polite"` on the position column only — not on every cell — to avoid screen reader noise.
- Print results page: position chart must include a `<figcaption>` with a text summary of position changes for accessibility.
