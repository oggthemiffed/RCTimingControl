## Layout Contract

### Pre-wizard gate (admin bootstrap)

- Centred card layout. No sidebar. No top nav.
- Single `Card` (max-width 400px, `mx-auto`) vertically centred on a full-viewport `bg-background` page.
- Card contains: page title "Set Up RC Timing" (`text-2xl font-semibold`), a subtitle "Create your admin account to get started" (`text-sm text-muted-foreground`), then the form fields.
- No header chrome, no footer. This screen appears before any user session exists.

### Wizard layout (5-step wizard at `/setup`)

- Mirrors `AdminPanelLayout` structure: fixed left sidebar (240px, `bg-card border-r`) + scrollable right content panel.
- **Sidebar** contains:
  - Brand block: "RC Timing — Setup" (`font-semibold text-base`), `px-4 py-5`.
  - `Separator`.
  - Step list (nav): 5 numbered items — see Step Sidebar Contract below.
  - `Separator`.
  - "Skip wizard" ghost link at bottom: `text-sm text-muted-foreground`, navigates to `/admin`.
- **Right panel**: `px-6 pt-6 pb-10`, `min-h-screen`, `md:pl-60`.
- Mobile behaviour: hamburger top-bar (same `h-14 sticky top-0` pattern as `AdminPanelLayout`); Sheet drawer for step list. No bottom nav on mobile — the wizard is an admin-only flow expected on a laptop.

### Completion summary page (final screen)

- Single-column layout within the right panel. No modal.
- Header: "Setup Complete" (`text-2xl font-semibold mb-2`) + "Your club is ready to run a meeting." (`text-sm text-muted-foreground mb-6`).
- Five `Card` components in a vertical stack (`space-y-4`), one per configured item, each with an edit link.
- Single primary CTA button at the bottom: "Go to Admin Panel".

## Step Sidebar Contract

Each step item in the wizard sidebar uses this visual language:

| State | Visual Treatment |
|-------|-----------------|
| Complete | `ri-checkbox-circle-fill` icon (remixicon), `text-flag-green`, step label at `text-sm font-medium`, full opacity |
| Current | `ri-record-circle-line` icon, `text-primary`, step label at `text-sm font-semibold`, left border accent `border-l-[3px] border-primary pl-[calc(0.75rem-3px)]` |
| Incomplete | `ri-circle-line` icon, `text-muted-foreground`, step label at `text-sm`, normal opacity |

Step list (ordered, not interactive links — wizard drives navigation via Next/Skip buttons):

1. Club Profile
2. Track
3. Race Format
4. Staff Account
5. Decoder Config

Step items are rendered as non-clickable list items (not NavLinks). Step navigation is linear — only the "Back" and "Next/Skip" buttons in the right panel change steps. Re-entry from Admin sidebar starts at the first incomplete step and allows free navigation via step items (clickable in re-entry mode).

## Interaction Contracts

### Admin bootstrap pre-gate

- Condition: `GET /api/v1/setup/status` returns `{ setupComplete: false }` AND zero users exist.
- Screen appears at `/setup` before the wizard sidebar.
- Fields: Email (`type="email"`), Password (`type="password"`), Confirm Password (`type="password"`).
- Zod schema: reuses the same password rules as `RegisterPage` (min 8 chars, confirmation match).
- Submit label: "Create Admin Account".
- On success: auto-login, wizard sidebar appears, navigate to Step 1 (Club Profile).
- Error state: inline `FormMessage` under offending field. Full-page error not used.

### Wizard step progression

- "Save and Continue" (primary): validates current step's form, POSTs/PATCHes to backend, advances to next step.
- "Skip for now" (ghost/outline): only on steps 2–5. Advances without saving. Step remains marked incomplete in sidebar.
- "Back" (ghost): returns to previous step. No save on back.
- URL does not encode step number. The wizard tracks current step in React state (and re-entry derives step from `GET /api/v1/setup/progress` booleans).
- Step 1 (Club Profile) has no "Skip" — club name is required to clear the first-run redirect.

### "Test connection" on Decoder Config step

- Button label: "Test Connection" (outline variant, full width below the decoder fields).
- On click: spinner replaces button text ("Testing…"); polls `GET /api/v1/race-control/forwarder/status` every 2 seconds.
- Timeout: 30 seconds. After 30s with no CONNECTED response, show Alert: "Forwarder not yet connected. Make sure the forwarder JAR is running with the downloaded config." Button returns to "Test Connection".
- On CONNECTED: replace button with Badge ("Connected", flag-green border/text). Show inline note: "Connection confirmed. You can proceed."
- Non-blocking: "Skip for now" remains available throughout the test.

### Token management on Decoder Config step

- If no token exists: "Generate Token" button (primary, full width).
- If token exists: show "Active" Badge (flag-green), generated-at timestamp. Offer "Regenerate" (outline) and inline confirmation pattern (matches `ForwarderTokenPage` exactly — muted background box, "Confirm Regenerate" / "Keep current token").
- One-time reveal: new token displayed in read-only mono Input with Copy button. Alert box: "Copy this token now — it will not be shown again."

### "Download forwarder.env" on Decoder Config step

- Button label: "Download forwarder.env" (outline variant, `ri-download-line` icon).
- Triggers `GET /api/v1/setup/forwarder-config-download` with `Content-Disposition: attachment`.
- Always generates fresh — button is always enabled if a token exists.
- If no token yet: button is disabled with tooltip "Generate a token first".

### Form validation

- All validation is client-side first (Zod via React Hook Form), server errors surfaced via toast (Sonner) for network/conflict errors.
- Required fields: red `FormMessage` below the field on blur or submit attempt. No inline red borders — rely on `FormMessage` text only (existing project pattern).
- Success actions: Sonner toast "Club profile saved" / "Track saved" / etc. after each step save.

## Accessibility Contract

- All interactive elements have `aria-label` where icon-only (matches existing codebase pattern).
- Wizard sidebar step items use `aria-current="step"` on the current step.
- Password inputs use `autocomplete="new-password"` on bootstrap form.
- "Test Connection" button has `aria-live="polite"` region for status updates.
- Minimum touch target: 44px height for all buttons (use `size="sm"` minimum from shadcn Button).
- Focus order: sidebar is `tabIndex={-1}` group; main form panel receives focus on step transition.
